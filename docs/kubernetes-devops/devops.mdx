## Azure DevOps Demo Scenarios
Azure devops demo will use
- Github repository to store the source code
- Azure devops to implement a "build & deployment" pipeline, a docker image will be created from demo application, then it gets pushed to azure container registry, eventually gets deployed to AKS

To use this demo, follow "Demo Steps" in below section

## Demo Steps
1. Open URL https://github.com/devopsonazure/vote-devops/blob/master/azure-vote/config_file.cfg, click edit icon(pen icon) then change TITLE from `Azure Voting App - Azure Devops Pipeline(v1)` to `Azure Voting App - Azure Devops Pipeline(v2)`
```txt {2}
# UI Configurations
TITLE = 'Azure Voting App - Azure Devops Pipeline(v1)'
VOTE1VALUE = 'Cats'
VOTE2VALUE = 'Dogs'
SHOWHOST = 'false'
```

2. Click `Commit changes` button to commit the change
<img alt="devops-2" src={useBaseUrl('img/kubernetes-devops/devops-2.png')}/>

3. Open URL https://dev.azure.com/pipelineonazure/vote-devops/_build, refresh the page (might be multiple times), until Azure devops pipeline is started.
<img alt="devops-3" src={useBaseUrl('img/kubernetes-devops/devops-3.png')}/>

4. This is a multi-stages pipeline, it contains `Build stage` and `Deploy stage`
<img alt="devops-4" src={useBaseUrl('img/kubernetes-devops/devops-4.png')}/>

5. Click build and deploy stage, show it to your customer how build & deploy stages look like
<img alt="devops-5" src={useBaseUrl('img/kubernetes-devops/devops-5.png')}/>
<img alt="devops-6" src={useBaseUrl('img/kubernetes-devops/devops-6.png')}/>

6. Once build & deploy are done (status changes to green), open URL http://vote-devops.k8s.devsecops.ink/, application's version should be changed to 'Azure Voting App - Azure Devops Pipeline(v2)'
<img alt="devops-7" src={useBaseUrl('img/kubernetes-devops/devops-7.png')}/>

7. If your customer is interested in how to write "Azure Devops Pipeline", you can open https://github.com/devopsonazure/vote-devops/blob/master/azure-pipelines.yml to show pipeline definition.
